Thanks for your interest in using these projects.

## Getting help
Support is primarily provided through GitHub issues and discussions in individual repositories.

Before opening a new issue:
- Check the README and documentation for the repository.
- Search existing issues to see if your question has already been answered.

## Bug reports and feature requests
- Use the issue templates provided in each repository.
- Include relevant details such as version information, environment, and reproduction steps.

## Security issues
Do not report security vulnerabilities in public issues.
Please follow the instructions in `SECURITY.md`.

## Scope of support
These projects are maintained on a best-effort basis.
We may not be able to provide:
- real-time support
- private troubleshooting
- custom development or consulting

## Other inquiries
For topics that don’t fit well into an issue or pull request, refer to the repository’s README or linked resources.
